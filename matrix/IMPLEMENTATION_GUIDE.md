**Scope:** Employee ID Login Overhaul + All Critical Bug Fixes

## What Was Changed & Why

### 1. Employee ID System (New — Primary Feature)

**Old:** Employees logged into Matrix with their email + password  
**New:** Employees log in with a **6-character Employee ID** + password

#### Employee ID Format
```
H3RK9N   7HR2K4   H2K9R3   4H9RK2
```
- Exactly **6 characters**
- Always contains **H** and **R** (Hospain HR branding)
- Mix of uppercase letters + digits
- No ambiguous characters (no O/0 confusion)
- Generated by `generate_employee_id()` in `auth_service.py`

#### How Employees Get Their ID
1. HR admin or Super Admin goes to **Matrix → Workforce → Employee Accounts**
2. Fills in name, email (optional), role
3. System generates a unique Employee ID + temporary password
4. Admin shares ID and temp password with the employee **securely** (phone call, in-person, encrypted message)

#### First Login Flow
```
Employee enters ID + temp password
         ↓
Login succeeds → API returns must_change_password: true
         ↓
Frontend FORCES redirect to /change-password
         ↓
Employee sets permanent password (min 8 chars, 1 uppercase, 1 number)
         ↓
New session token issued → Redirected to Command Center
```

Employee can also change password any time from **Settings → Change Password**.

### 2. Files Changed

#### Backend: `auth-service/app/models/user.py`
- Added `employee_id` column (VARCHAR 10, unique, indexed)
- Added `is_temporary_password` column (BOOLEAN, default FALSE)
- Added Hospain internal roles to `RoleEnum`: manager, team_lead, l1, l2, support, finance, engineering, onboarding, data, verification, employee

#### Backend: `auth-service/app/api/v1/auth.py`
- **`POST /api/v1/auth/login`** — Now accepts `employee_id` as primary identifier. Returns `must_change_password: true` when temp password is active
- **`POST /api/v1/auth/change-password`** — New endpoint. Clears `is_temporary_password`, bumps `token_version` (invalidates old sessions), returns fresh token
- **`POST /api/v1/auth/employees/create`** — New endpoint (admin/hr only). Generates Employee ID + temp password, creates account
- **`GET /api/v1/auth/employees/validate-id/{id}`** — Validates Employee ID exists (for UX)
- **`GET /api/v1/auth/run-auth-migrations`** — Now requires `X-Migration-Secret` header (BUG-9 fix)
- **BUG FIX:** All 8 `await db.flush()` calls now followed by `await db.commit()` (was silently losing data on process restart)
- **Migration endpoint** now includes `employee_id` and `is_temporary_password` columns

#### Backend: `auth-service/app/middleware/rbac.py`
- **BUG-2 FIX:** Changed `from jose import JWTError` → `import jwt as pyjwt` — python-jose was causing 500 errors on every authenticated request
- **BUG-3 FIX:** Token claim `hid` → `hospital_id` (JWT writes `hospital_id`, RBAC was reading `hid` — always empty)
- **BUG-4 FIX:** Token claim `ver` → `token_version` (JWT writes `token_version`, RBAC was reading `ver` — always 0)
- **BUG-19 FIX:** Role check now uses exact equality (not substring) — `"super_employee"` no longer matches `"employee"`
- Added `employee_id` and `must_change_password` to `CurrentUser` model

#### Backend: `auth-service/app/core/security.py`
- **BUG-7 FIX:** `get_jwks()` was calling `pub_key.public_key().public_numbers()` — `_PUBLIC_KEY` is already a public key, `.public_key()` doesn't exist on it. Fixed to `pub_key.public_numbers()` directly

#### Backend: `auth-service/app/services/auth_service.py`
- Added `generate_employee_id()` — cryptographically random 6-char ID always containing H+R
- Added `generate_temp_password()` — strong temp password meeting complexity requirements

#### Backend: `auth-service/alembic/versions/003_employee_id_temp_password.py`
- New migration adding `employee_id` and `is_temporary_password` columns
- Adds missing Hospain internal roles to the `roleenum` PostgreSQL enum type
- Safe to run multiple times (IF NOT EXISTS guards)

#### Matrix Frontend: `src/pages/Login.jsx`
- Replaced email/phone input with **Employee ID** input
- Added format validation (6 chars, contains H and R)
- **BUG-31 FIX:** Error detection by message content, not fragile string-includes-'401'
- Duplicate role in ALLOWED_ROLES removed
- **BUG-19 FIX:** Role check uses `Set.has()` (exact equality, not substring)
- Password show/hide toggle added

#### Matrix Frontend: `src/pages/ChangePassword.jsx` (NEW)
- Full-screen forced password change on first login
- Password rule display (live validation as user types)
- Confirm password mismatch detection
- Issues fresh session token on success, old token invalidated
- "Sign out" option if wrong person logged in

#### Matrix Frontend: `src/pages/matrix/EmployeeAccounts.jsx` (NEW)
- HR admin tool to create employee accounts
- Shows generated Employee ID in large monospace display
- Shows temp password with copy button
- Clear instructions for giving credentials to employee

#### Matrix Frontend: `src/components/ProtectedRoute.jsx`
- **Intercepts `must_change_password`** — any route redirect to `/change-password`
- **BUG-23 FIX:** Session stored in `sessionStorage` only (not `localStorage`) — no cross-tab stale state

#### Matrix Frontend: `src/App.jsx`
- Added `/change-password` route (public — accessible before auth completes)
- Added `matrix/employee-accounts` route (HR-gated)

### 3. Database Migration

Run this **once** on the production database:

```bash
# Option A: Alembic (recommended)
cd backend/auth-service
alembic upgrade head

# Option B: Manual endpoint (requires MIGRATION_SECRET header)
curl -H "X-Migration-Secret: YOUR_SECRET" https://your-api/api/v1/auth/run-auth-migrations
```

### 4. Environment Variables Required

Add to `.env` / GCP Secret Manager:

```bash
# REQUIRED for production JWT security (generate once, keep forever)
JWT_PRIVATE_KEY_PEM=<base64-encoded PEM>
JWT_PUBLIC_KEY_PEM=<base64-encoded PEM>

# How to generate:
# openssl genrsa -out private_key.pem 2048
# openssl rsa -in private_key.pem -pubout -out public_key.pem
# base64 -w0 private_key.pem  → JWT_PRIVATE_KEY_PEM
# base64 -w0 public_key.pem   → JWT_PUBLIC_KEY_PEM

# REQUIRED for migration endpoint security
MIGRATION_SECRET=<random-32-char-string>

# REQUIRED for OTP delivery (employees don't use OTP but patients/doctors do)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=noreply@hospain.in
SMTP_PASSWORD=<app-password>
SMTP_FROM_EMAIL=noreply@hospain.in
```

### 5. Employee ID Examples

```
H3RK9N   ← H at pos 0, R at pos 2
7HR2K4   ← H at pos 1, R at pos 2
4H9RK2   ← H at pos 1, R at pos 3
K2H9R7   ← H at pos 2, R at pos 4
```

All are exactly 6 characters. The H and R positions are randomised each time. This gives ~720 possible H+R position combinations × many digit/letter combinations = millions of unique IDs.
